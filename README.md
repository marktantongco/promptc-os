## What is promptc OS?

promptc OS is a production-grade prompt engineering environment that treats prompt creation as a systematic engineering discipline. It provides structured tools across 6 interconnected zones — from initial activation through building, validation, playbook orchestration, monetization, and system management.

Every prompt, modifier, template, and workflow in the system is **copy-ready** — paste directly into ChatGPT, Claude, Gemini, or any AI chat. The universal Collection Basket (⌘B) lets you gather your favorite prompts across all zones, the Clipboard History tracks every copy action, and the Command Palette (⌘K) provides instant search across the entire library.

### What's New in v3.7

## Key Features

### 🔧 Modifier Assembly
- User input field for base prompt at the top of Modifiers tab
- **+Add** button on every modifier to add it to the assembly
- Numbered assembly list with individual remove buttons
- **Copy Assembly** exports the complete numbered modifier chain
- Formats modifiers for sequential prompt connection and anticipation

### 🐾 Flexible Animal Selection
- **Multi-select** animals with checkbox toggles
- Selected animals highlighted with colored border
- **Combined Generation** — generate a mega-prompt from any combination of animals
- User input integrates with combined animal prompts
- Selection state persists across sessions via localStorage

### 🛠️ Prompt Enhancer
- User input field at the top of Enhancements tab
- **Checkboxes** on each enhancement tool to select which to apply
- **Generate Enhanced** combines base prompt with all selected enhancement techniques
- Enhanced result displayed with copy button
- Tool count indicator shows how many are selected

### 📋 Clipboard History
- **Every copy action** is tracked with timestamp and zone
- Slide-in panel (like basket) with full history
- **Copy All** button for bulk clipboard export
- **Clear** button to reset history
- **Individual re-copy** on any historical item
- Persists up to 50 items across sessions

### ➕ Universal +Add to Basket
Available on items from: **Modifiers, Lint Rules, Word Swaps, Vocabulary, Typography, Design Combos, Principles**
- Each item gets a **+Add** button (Plus icon) alongside the copy button
- Items are formatted with zone prefix and full content when added
- All added items flow into the existing basket system

### 🧺 Collection Basket (⌘B)
- Collect prompts from any zone — click copy and it auto-adds to basket
- **Pin** important items, **Favorite** with star indicator
- **Multi-select** with batch copy, remove, and export
- **Sort** by newest, oldest, longest, shortest, A→Z
- **Search** and filter by zone, **duplicate detection**
- **Export** to markdown (.md) and JSON formats
- **Pipeline progress** visualization across zones
- **Smart recommendations** based on basket activity
- **Stats & Insights** tabs with zone distribution analytics
- **localStorage persistence** — survives page reloads (up to 100 items)
- **Cross-zone forwarding** — send items to Build, Validate, or Activate zones

### 🔍 Command Palette (⌘K)
- Fuzzy search across all modifiers, templates, tasks, workflows, brands, animals, enhancements, lint rules, word swaps, and chains
- Zone quick-switch grid, keyboard navigation (arrows + Enter)
- Recently copied items section

## Getting Started

### Prerequisites
- Node.js 18+ or Bun
- A Vercel account (for deployment)

### Local Development
```bash
# Clone the repository
git clone https://github.com/marktantongco/promptc-os.git
cd promptc-os

# Install dependencies
bun install

# Start development server
bun run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

### Deploy to Vercel
```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel
```

Or connect your GitHub repo directly at [vercel.com/new](https://vercel.com/new).

### GitHub Pages
The app is also available at [marktantongco.github.io/promptc-os](https://marktantongco.github.io/promptc-os/).

## Operating Principles

1. **NO ONE-OFF WORK** — Every task produces a reusable asset
2. **THE RULE** — Plan → Validate → Execute, never skip validation
3. **PLAN → VALIDATE → EXECUTE** — Break down, test assumptions, then build
4. **COMPOUNDING SYSTEM** — Build once → runs forever → every skill makes the system smarter
5. **SKILL FORMAT** — 4-section SKILL.md: Context, Instructions, Constraints, Examples
6. **SECURITY RULES** — No hardcoded secrets, validate inputs, rate limit by default
